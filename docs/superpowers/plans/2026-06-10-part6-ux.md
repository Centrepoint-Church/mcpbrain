# Part 6 — UX: State-Aware Home, Menu Bar States, Self-Healing — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `/` a state-aware home (onboarding wizard when unconfigured → live status/control center when configured), give the menu-bar tray real icon/title states plus a contextual "Reconnect" action, and render every broken connection as a plain-language self-healing banner — all off the `status()` `connections` block from Plan 4.

**Architecture:** The browser home (`wizard/index.html`) gains a **status-center view** shown when `status().is_configured` is true (the existing 5-step wizard is the unconfigured/empty state). The status center renders `status().connections` as cards (state + a one-click fix), `status().backfill` as progress, and an "Enrich history with Claude Code" button that calls the Plan 7 endpoints. The tray (`TrayController`) derives an icon state and an attention line from `connections`, adds a contextual "Reconnect Google" menu item and the version, and fires an OS notification when a connection newly needs action. **No new control-API routes are needed** — reconnect reuses `POST /api/auth/start`; enrich-history uses Plan 7's `/api/enrich-backfill/*`.

**Tech Stack:** Python 3.12, pytest (tray logic + served-HTML element presence). HTML/JS rendering is wired against the existing `__MCPBRAIN_TOKEN__`-injected fetch helpers.

This is **Plan 6 of the productization series** — spec **§3.1, §3.3, §3.4, §3.5**. It depends on **Plan 4** (`status().connections`) and **Plan 7** (the enrich-backfill endpoints + `status().backfill`).

**Grounding (verified):**
- `status()` returns `is_configured`, `connections` (Plan 4), `backfill` (Plan 7), `paused`, `chunk_count`, `enriched_count`, `google_connected`, `google_account`, `open_findings`, `spool`.
- `TrayController` (tray.py): `status_text()`, `menu_items() -> list[(label, handler, enabled)]`, `is_paused()`, `review_count()`, `on_pause/on_resume/on_open_setup/on_open_dashboard/on_quit`, `_last` is the cached status dict. `run_tray` loops every 3s: `refresh()`, sets `icon.title`, `icon.update_menu()`, and `icon.notify(...)` when `review_count()` rises.
- `ControlClient` (control_client.py): `status()`, `pause()`, `resume()`, `wizard_url()`, `dashboard_url()`. (Add `reconnect_google()` / `start_enrich_backfill()` POST helpers.)
- `_serve_wizard` serves `wizard/index.html` with `__MCPBRAIN_TOKEN__` replaced. The page's JS uses `H` (auth header), `P(path, body)` (POST), `$ (id)`, `renderStatus(j)`, `refresh()` (polls `/api/status` every 3s).
- Tests: `tests/test_tray.py` uses a `FakeClient` with a `status()` dict; `tests/test_wizard_serve.py` asserts element ids are present in the served HTML.

```python
    def attention(self) -> list[dict]:
        """Connections that need the user to act, as [{connection, detail}]."""
        if not self._last:
            return []
        conns = self._last.get("connections") or {}
        return [{"connection": name, "detail": c.get("detail", "")}
                for name, c in conns.items() if c.get("state") == "needs_action"]

    def icon_state(self) -> str:
        """One of: unavailable | paused | attention | running."""
        if not self._available or self._last is None:
            return "unavailable"
        if self.attention():
            return "attention"
        if self._last.get("paused"):
            return "paused"
        return "running"
```

```javascript
function renderHome(j){
  const configured = !!j.is_configured;
  $("home-status").classList.toggle("hidden", !configured);
  // (show the wizard steps only when NOT configured — toggle their container)
  document.querySelectorAll(".wizard-step").forEach(s => s.classList.toggle("hidden", configured));
  if(!configured) return;
  $("hs-version").textContent = j.version ? ("v" + j.version) : "";
  $("hs-health").textContent = j.paused ? "Paused" : "Running";
  renderConnections(j.connections || {});
  renderBackfill(j.backfill || {});
}

function renderConnections(conns){
  const order = ["google","claude","clickup","backup","records"];
  const cards = order.filter(k => conns[k]).map(k => {
    const c = conns[k];
    const fix = (k === "google" && c.state === "needs_action")
      ? ' <button onclick="reconnectGoogle()">Reconnect</button>' : "";
    return `<div class="conn ${c.state}"><b>${k}</b>: ${c.state} — ${c.detail||""}${fix}</div>`;
  }).join("");
  $("conn-cards").innerHTML = cards;
  // Self-healing banners: any needs_action connection
  const banners = order.filter(k => conns[k] && conns[k].state === "needs_action")
    .map(k => `<div class="banner">${conns[k].detail||k}` +
              (k==="google"?' <button onclick="reconnectGoogle()">Reconnect</button>':"") + `</div>`).join("");
  $("self-heal-banners").innerHTML = banners;
}

function renderBackfill(bf){
  const parts = Object.keys(bf).map(src => {
    const s = bf[src];
    return `${src}: ` + (s.done ? "history complete" : ("indexing… " + (s.reached || "recent")));
  });
  $("backfill-progress").textContent = parts.join(" · ");
}

async function reconnectGoogle(){ try{ await P("/api/auth/start"); }catch(e){} }

$("enrich-history-btn").addEventListener("click", async () => {
  $("enrich-state").textContent = "Starting…";
  try{ await P("/api/enrich-backfill/start"); $("enrich-state").textContent = "Running in background…"; }
  catch(e){ $("enrich-state").textContent = "Could not start"; }
});
```
